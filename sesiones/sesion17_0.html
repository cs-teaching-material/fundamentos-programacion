Concepto A — Firmas correctas

La firma define como se usa la funcion

/// <summary>Calcula el area de un rectangulo.</summary> /// <param name="baseRectangulo">Base del rectangulo.</param> /// <param name="altura">Altura del rectangulo.</param> /// <returns>Area calculada.</returns> static double CalcularArea(double baseRectangulo, double altura) { return baseRectangulo * altura; }

Reglas practicas

  • El nombre debe iniciar con verbo.
  • Cada parametro debe aportar una sola idea.
  • El retorno debe resolver una necesidad concreta.
  • Evita preguntar por consola dentro de la funcion de calculo.

La firma debe permitir entender la funcion sin leer todo el codigo.

Concepto A — Flujo de datos

Entradas por parametro, salida por retorno

Lectura

  • Los parametros entran a la funcion.
  • La funcion transforma esos datos.
  • El retorno vuelve a quien llamo.

Si el retorno no sirve en Main, probablemente la firma esta mal pensada.

Estrategia — Descomponer antes de codar

Preguntas clave antes de escribir funciones

¿Cuantos pasos tiene?

  • Leer entrada
  • Validar datos
  • Calcular resultado
  • Mostrar salida

Cada uno puede ser una funcion.

¿Cual es la transformacion?

  • ¿Que entra? (parametros)
  • ¿Que ocurre adentro?
  • ¿Que sale? (retorno)

Si no es claro, la funcion es muy compleja.

Regla: Si escribes una funcion sin antes pensar en sus responsabilidades, probablemente sea demasiado grande.

Jerarquia simple — Una funcion llama otra

Construir nivel por nivel

// Nivel 1: entrada y logica basica static double ObtenerPeso() { Console.WriteLine("Peso en kg:"); return double.Parse(Console.ReadLine()); } // Nivel 2: calculo puro static double CalcularIMC(double peso, double altura) { return peso / (altura * altura); } // Nivel 3: orquestacion (usa nivel 1 y 2) static void CalcularYMostrar() { double peso = ObtenerPeso(); double imc = CalcularIMC(peso, 1.75); Console.WriteLine("IMC: " + imc); }

Las funciones de nivel superior reutilizan funciones de nivel inferior. Esto es composicion.

Alcance — Variables locales vs parametros

Donde vive cada variable

Variable local (dentro de funcion)

static void Saludar() { string nombre = "Ana"; Console.WriteLine(nombre); // nombre solo existe aqui } // nombre NO existe aqui

Parametro (entra a la funcion)

static void Saludar(string nombre) { Console.WriteLine(nombre); // nombre existe aqui } // nombre NO existe aqui

Las variables declaradas dentro de una funcion no existen fuera. Esto se llama scope o alcance.

Ejemplo 1 — Calculadora de precio

Separar entrada, calculo y salida

/// <summary>Calcula el IVA.</summary> static double CalcularIVA(double subtotal) { return subtotal * 0.19; } /// <summary>Calcula el total.</summary> static double CalcularTotal(double subtotal, double iva) { return subtotal + iva; }

Uso en Main

static void Main() { Console.WriteLine("Precio:"); double subtotal = double.Parse(Console.ReadLine()); double iva = CalcularIVA(subtotal); double total = CalcularTotal(subtotal, iva); Console.WriteLine("Total: " + total); }

Ejemplo 2 — Calculo de calificacion

Parametros multiples y logica clara

/// <summary>Calcula promedio de 3 calificaciones.</summary> /// <param name="n1">Calificacion 1.</param> /// <param name="n2">Calificacion 2.</param> /// <param name="n3">Calificacion 3.</param> /// <returns>Promedio calculado.</returns> static double CalcularPromedio(double n1, double n2, double n3) { double promedio = (n1 + n2 + n3) / 3; return promedio; }

Validacion

Si una funcion recibe datos inciertos, valida ANTES de procesar.

static bool EsNotaValida(double nota) { return nota >= 0 && nota <= 5; }

Ejemplo 3 — Conversion de temperaturas

Multiples funciones coordinadas

static double CelsiusAFahrenheit(double celsius) { double fahrenheit = (celsius * 9 / 5) + 32; return fahrenheit; } static bool EsTemperaturaSegura(double celsius) { if (celsius >= 0 && celsius <= 40) { return true; } return false; }
static void Main() { Console.WriteLine("Temperatura en Celsius:"); double temp = double.Parse(Console.ReadLine()); double fahr = CelsiusAFahrenheit(temp); bool segura = EsTemperaturaSegura(temp); Console.WriteLine("Fahrenheit: " + fahr); Console.WriteLine("Segura: " + segura); }

Ejemplo 4 — Descuentos segun tipo

Decisiones dentro de funciones de logica

/// <summary>Calcula descuento segun cliente.</summary> /// <param name="tipo">Tipo de cliente: normal, vip o estudiante.</param> /// <param name="subtotal">Monto sin descuento.</param> /// <returns>Descuento aplicable.</returns> static double CalcularDescuento(string tipo, double subtotal) { double porcentaje = 0; if (tipo == "vip") { porcentaje = 0.15; } else if (tipo == "estudiante") { porcentaje = 0.10; } double descuento = subtotal * porcentaje; return descuento; }

Ejemplo 5 — Uso de ref

Cuando necesitas modificar el original

static void Incrementar(ref double cantidad) { cantidad = cantidad + 1; } double x = 5; Incrementar(ref x); Console.WriteLine(x);

Resultado

Salida: 6

Sin ref, x seguiria siendo 5.

Usa ref solo cuando la funcion necesita cambiar realmente el valor externo.

Ejemplo 6 — Uso de out

Para devolver un valor adicional

/// <summary>Intenta parsear una nota.</summary> static bool LeerNotaValida(string texto, out double nota) { nota = 0; bool esValido = double.TryParse(texto, out nota); if (esValido && nota >= 0 && nota <= 5) { return true; } return false; } static void Main() { Console.WriteLine("Nota:"); if (LeerNotaValida(Console.ReadLine(), out double n)) { Console.WriteLine("Nota aceptada: " + n); } }

Comparacion — Mal vs Bien

Errores frecuentes en separacion UI / logica

❌ Incorrecto

static double Promedio() { Console.WriteLine("Nota 1:"); double n1 = double.Parse( Console.ReadLine() ); Console.WriteLine("Nota 2:"); double n2 = double.Parse( Console.ReadLine() ); return (n1 + n2) / 2; }

Mezcla lectura con calculo.

✅ Correcto

static double Promedio(double n1, double n2) { return (n1 + n2) / 2; } static void Main() { Console.WriteLine("Nota 1:"); double nota1 = double.Parse( Console.ReadLine() ); Console.WriteLine("Nota 2:"); double nota2 = double.Parse( Console.ReadLine() ); double prom = Promedio(nota1, nota2); }

Logica pura, reutilizable.

Composicion — Funciones que usan funciones

Construir soluciones modulares

static double CalcularSubtotal(int cantidad, double precioUnitario) { return cantidad * precioUnitario; } static double CalcularIVA(double subtotal) { return subtotal * 0.19; } static double CalcularTotal(int cantidad, double precioUnitario) { double subtotal = CalcularSubtotal(cantidad, precioUnitario); double iva = CalcularIVA(subtotal); return subtotal + iva; }

Nota: CalcularTotal reutiliza las otras funciones en lugar de repetir logica.

Checklist — ¿Listo?

Antes de dar por lista una funcion, verifica

  • ✓ La firma comienza con verbo: CalcularPromedio, EsValido, Convertir.
  • ✓ Los parametros tienen nombres significativos, no p, n, x.
  • ✓ Documentada con XML summary, param, returns.
  • ✓ No hace Console.ReadLine en funciones de logica.
  • ✓ Si retorna algo, lo usa quien la llama.
  • ✓ Los nombres de parametro coinciden con XML.
  • ✓ Probada con datos conocidos.
