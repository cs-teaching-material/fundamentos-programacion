Concepto A — Criterios para parametros

¿Que va como parametro?

Va como parametro todo dato que la funcion necesita para hacer su trabajo y que no puede ni debe obtener por si misma: valores numericos, textos, banderas booleanas, listas.

No va como parametro (ni debe leerse dentro de la funcion): la entrada del usuario, la configuracion global que todavia no existe, ni constantes que la funcion puede declarar internamente.

Regla practica

Preguntate: "Si usara esta funcion en otro programa, ¿que informacion necesitaria pasarle?"

  • Calculo de impuesto → parametro: el monto base.
  • Validacion de rango → parametros: el valor, min, max.
  • Formato de salida → parametros: el dato ya calculado.

Concepto A — Paso por valor

El comportamiento predeterminado: copia

static void Duplicar(int n) { n = n * 2; // modifica la copia local Console.WriteLine($"Dentro: {n}"); } static void Main() { int valor = 5; Duplicar(valor); Console.WriteLine($"Fuera : {valor}"); // Fuera : 5 ← valor no cambia }

Que ocurre

  • C# copia el valor al parametro n.
  • Cualquier cambio a n dentro de la funcion no afecta la variable original.
  • Es el comportamiento por defecto para tipos de valor (int, double, bool, char).

En la mayoria de funciones de calculo, el paso por valor es lo correcto.

Concepto A — Paso por referencia

ref y out: cuando la funcion debe modificar la variable

ref — leer y modificar

static void Incrementar(ref int n) { n++; } int x = 3; Incrementar(ref x); // x es ahora 4

La variable debe estar inicializada antes de pasarla con ref.

out — solo escribir

static bool TryParsearNota( string entrada, out double nota) { return double.TryParse(entrada, out nota); } if (TryParsearNota(Console.ReadLine(), out double n)) Console.WriteLine($"Nota: {n}");

out: no requiere inicializacion previa; la funcion asigna el valor.

Regla de uso: evita ref / out por defecto. Usalos solo cuando la firma con valor de retorno normal no es suficiente (ej. retornar dos resultados distintos).

Concepto B — Documentacion XML

Comentarios XML: el contrato de la funcion

/// <summary> /// Calcula el promedio de tres notas. /// </summary> /// <param name="a">Primera nota (0.0 - 5.0).</param> /// <param name="b">Segunda nota (0.0 - 5.0).</param> /// <param name="c">Tercera nota (0.0 - 5.0).</param> /// <returns>Promedio de las tres notas.</returns> static double CalcularPromedio(double a, double b, double c) { return (a + b + c) / 3; }

Por que documentar

  • VS Code muestra el tooltip de IntelliSense al escribir el nombre de la funcion.
  • El equipo (o tu futuro yo) entiende el contrato sin leer la implementacion.
  • Herramientas como DocFX generan documentacion HTML automaticamente.

Escribe /// sobre cualquier funcion en VS Code: genera el esqueleto automaticamente.

Practica guiada — Flujo de ejecucion

Que hace C# cuando llamas una funcion

Paso 1: Main lee las notas del usuario y las almacena en variables locales.
Paso 2: Main llama a CalcularPromedio(n1,n2,n3). C# copia los valores a los parametros a, b, c.
Paso 3: CalcularPromedio ejecuta su logica y retorna el double. El control vuelve a Main.
Paso 4: Main recibe el resultado, llama EsAprobado y finalmente MostrarReporte.

Pila de llamadas (call stack)

Cada llamada agrega un marco. Al retornar, el marco desaparece.

Reflexion — Preguntas de comprension

Verifica tu entendimiento

Pregunta 1: Una funcion CalcularDescuento(double precio) retorna el precio final despues del 10 % de descuento. ¿El precio original en Main cambia despues de llamarla?

Respuesta: No. Es paso por valor; la variable de Main no cambia.

Pregunta 2: ¿Cuando usarias out en lugar de un return normal?

Respuesta: cuando la funcion necesita "devolver" mas de un resultado distinto, o para seguir el patron de TryParse.

Pregunta 3: ¿Que ventaja inmediata obtienes al escribir /// sobre una funcion en VS Code?

Respuesta: IntelliSense muestra el tooltip con resumen, parametros y retorno al escribir el nombre de la funcion en cualquier lugar del codigo.

Caso: consumo electrico modular y documentado

Meta de demostracion: construir en vivo un programa en C# que reciba watts, horas y tarifa; luego calcule consumo diario, consumo mensual y costo mensual usando funciones con firma clara y documentacion XML.

Entradas y salidas

  • Entradas: watts, horasDia, precioKWh.

Regla didactica

Paso 3 — Implementar funciones de calculo

Logica pura: sin lectura ni impresiones

/// <summary>Calcula consumo diario en kWh.</summary> /// <param name="watts">Potencia en watts.</param> /// <param name="horasDia">Horas de uso diario.</param> /// <returns>Consumo diario en kWh.</returns> static double ConsumoDiario(double watts, double horasDia) { return watts * horasDia / 1000.0; } /// <summary>Calcula consumo mensual suponiendo 30 dias.</summary> /// <param name="consumoDiario">Consumo diario en kWh.</param> /// <returns>Consumo mensual en kWh.</returns> static double ConsumoMensual(double consumoDiario) { return consumoDiario * 30.0; } /// <summary>Calcula costo mensual.</summary> /// <returns>Costo total en moneda local.</returns> static double CostoMensual(double consumoMes, double precioKWh) { return consumoMes * precioKWh; }

Chequeo rapido: para 1200 W y 5 h/dia, ConsumoDiario debe retornar 6.0.

Paso 4 — Captura segura en Main

Validar entrada sin ensuciar la logica de calculo

static bool LeerDouble(string mensaje, out double valor) { Console.Write(mensaje); return double.TryParse(Console.ReadLine(), out valor); } static void Main() { if (!LeerDouble("Watts: ", out double watts) || !LeerDouble("Horas por dia: ", out double horas) || !LeerDouble("Precio por kWh: ", out double tarifa)) { Console.WriteLine("Entrada invalida. Usa numeros."); return; } // siguiente paso: llamar funciones de calculo }

Por que si usamos out aqui

  • El metodo retorna dos cosas: exito (bool) y valor leido (double).
  • Replica el patron TryParse del framework.
  • Mantenemos el manejo de entrada concentrado en una sola funcion.

Programa final integrado

using System; class Program { /// <summary>Calcula consumo diario en kWh.</summary> static double ConsumoDiario(double watts, double horasDia) { return watts * horasDia / 1000.0; } /// <summary>Calcula consumo mensual suponiendo 30 dias.</summary> static double ConsumoMensual(double consumoDiario) { return consumoDiario * 30.0; } /// <summary>Calcula costo mensual.</summary> static double CostoMensual(double consumoMes, double precioKWh) { return consumoMes * precioKWh; } static bool LeerDouble(string mensaje, out double valor) { Console.Write(mensaje); return double.TryParse(Console.ReadLine(), out valor); } static void Main() { if (!LeerDouble("Watts: ", out double watts) || !LeerDouble("Horas por dia: ", out double horas) || !LeerDouble("Precio por kWh: ", out double tarifa)) { Console.WriteLine("Entrada invalida. Usa numeros."); return; } double diario = ConsumoDiario(watts, horas); double mensual = ConsumoMensual(diario); double costo = CostoMensual(mensual, tarifa); Console.WriteLine("Diario: " + diario + " kWh | Mensual: " + mensual + " kWh | Costo: $" + costo); } }

Demostracion — Ajustes y extensiones

Como ampliar sin romper la firma

Mejora 1: redondeo y formato

  • Si quieres menos decimales, primero muestra el valor tal como sale del calculo.
  • El redondeo puede explicarse despues como tema aparte de biblioteca estandar.
  • Evitar convertir a int antes de terminar calculos.

Mejora 2: mas de un dispositivo

  • Crear una funcion acumuladora para sumar consumos diarios.
  • Reusar ConsumoMensual y CostoMensual sin cambios.
  • Ejemplo de modularidad: se agrega funcionalidad sin reescribir todo.

Mensaje clave: una firma bien pensada facilita mantenimiento, pruebas y extensiones.

Herramienta — VS Code + IntelliSense

Como ver el tooltip XML en VS Code

Pasos

  1. Escribe /// encima de tu funcion → VS Code genera el esqueleto XML.
  2. Completa <summary>, <param> y <returns>.
  3. En otro lugar del codigo, escribe el nombre de la funcion.
  4. IntelliSense muestra el tooltip con toda la documentacion.

Extension requerida

Extension: ms-dotnettools.csharp

Verificar que esta instalada en VS Code: Extensions → buscar "C#" → publicada por Microsoft.

La documentacion XML tambien funciona con dotnet doc y DocFX para generar sitios de documentacion.
